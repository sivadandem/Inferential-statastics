Probability Foundations
The language every statistical test speaks
🎲 1.1 — What is Probability? CONCEPT

Probability answers one question: How likely is something to happen?

It's always a number between 0 (impossible) and 1 (certain).

P(Event) = (Favorable outcomes) ÷ (Total possible outcomes)
🚢
Titanic Example:
891 passengers total. 342 survived.
P(Survived) = 342 ÷ 891 = 0.384 = 38.4%
This means: if you randomly picked one passenger, there's a 38.4% chance they survived.
Rules to Know
  • 0 ≤ P(A) ≤ 1 always
  • P(impossible) = 0
  • P(certain) = 1
  • Sum of all outcomes = 1
DA Usage
  • Survival probability per group
  • Churn probability
  • Click probability (A/B test)
  • Defect probability
🗂️ 1.2 — Types of Events CONCEPT
Event TypeMeaningExample (Titanic)
Independent One event doesn't affect the other P(Passenger survives) doesn't affect P(next passenger survives)
Dependent One event changes the probability of the other P(Survived | Female) — gender affects survival
Mutually Exclusive Can't happen at the same time A passenger can't be both Survived=1 and Survived=0
Exhaustive Together they cover all possibilities Survived + Not Survived = all passengers
🔗 1.3 — Conditional Probability CRITICAL

Probability of A given that B has already happened.

P(A | B) = P(A and B) ÷ P(B)

Read as: "Probability of A given B"

🚢
Titanic Example:
What's the probability of surviving given you are female?

P(Survived | Female) = (females who survived) ÷ (total females)
= 233 ÷ 314 = 0.742 = 74.2%

Compare to overall survival of 38.4% — being female changed the probability dramatically!
🧠
Why this matters for DA: Every time you do pd.crosstab(normalize='index'), you are calculating conditional probability! That 74% female survival rate you found in Phase 3B — that was P(Survived | Female).
⚡ 1.4 — Bayes' Theorem IMPORTANT

Bayes' Theorem lets you update your belief about something when you get new evidence.

P(A | B) = [ P(B | A) × P(A) ] ÷ P(B)
Terms:
P(A) = Prior probability (before evidence)
P(A|B) = Posterior (after evidence)
P(B|A) = Likelihood
P(B) = Normalizing factor
Titanic Example:
You know someone survived. What's the probability they were female?

P(Female | Survived) = P(Survived | Female) × P(Female) ÷ P(Survived)
= (0.742 × 0.352) ÷ 0.384 = 0.68
📍
Where DA uses Bayes: Spam filters, recommendation engines, Naive Bayes classifier, A/B test evaluation, medical diagnosis models. Core concept in ML classification.
📊 1.5 — Expected Value CONCEPT

The average outcome you'd expect if you ran the experiment many times.

E(X) = Σ [ x × P(x) ]
🚢
Titanic Example:
A random passenger: survived with P=0.384, died with P=0.616
E(Survived) = (1 × 0.384) + (0 × 0.616) = 0.384
This equals the mean of Survived column — expected value IS the mean for binary data!

Probability Distributions
The shapes that data takes — and why those shapes matter
🎯
What is a distribution? It tells you — for each possible value of your data — how likely that value is. Think of it as a map of your data's probability.
🔔 2.1 — Normal Distribution (Gaussian) MOST IMPORTANT
When
  • Continuous data
  • Data is symmetric around mean
  • Most values cluster in center
Where (DA)
  • Checking if Age is normal
  • Before running T-test
  • Outlier detection (3σ rule)
Why
  • Most statistical tests assume it
  • Central Limit Theorem relies on it
  • Standardization built on it
How to Identify
  • Bell-shaped histogram
  • Skewness ≈ 0
  • Mean ≈ Median ≈ Mode
Bell curve — symmetric around mean. 68% within 1σ, 95% within 2σ, 99.7% within 3σ
Key properties: Mean = Median = Mode  |  Defined by μ (mean) and σ (std dev)
🚢
Titanic: Age (skew=0.53) is approximately normal → T-test is valid for Age.
Fare (skew=4.78) is NOT normal → Use Mann-Whitney U instead of T-test for Fare.
python from scipy.stats import norm # What % of passengers have age between 20 and 40? mean_age = df['Age'].mean() # 29.1 std_age = df['Age'].std() # 13.0 prob = norm.cdf(40, mean_age, std_age) - norm.cdf(20, mean_age, std_age) print(f"P(20 < Age < 40) = {prob:.3f}") # ≈ 0.53
🎯 2.2 — Binomial Distribution VERY COMMON IN DA

Used when each trial has exactly two outcomes: success or failure.

When
  • Fixed number of trials (n)
  • Each trial: Yes or No
  • Same probability each time
Where (DA)
  • Survived = 1 or 0
  • Clicked ad = yes/no
  • Churned = yes/no
  • A/B testing
Why
  • Models binary outcomes
  • Foundation of logistic regression
  • A/B test significance
How to Identify
  • Target column is 0/1
  • Counting successes in n trials
P(X = k) = C(n,k) × p^k × (1-p)^(n-k)

n = total trials, k = desired successes, p = probability of success each trial

python from scipy.stats import binom # Survival rate from Titanic = 38.4% # What's the probability exactly 4 out of 10 random passengers survive? p = binom.pmf(k=4, n=10, p=0.384) print(f"P(exactly 4 survive) = {p:.4f}") # What's the probability AT LEAST 5 survive? p_atleast5 = 1 - binom.cdf(k=4, n=10, p=0.384) print(f"P(≥5 survive) = {p_atleast5:.4f}")
🔗
Connection to DA: When you do A/B testing ("did version B get more clicks?"), you're using binomial probability to check if the difference is real or chance.
⚡ 2.3 — Poisson Distribution GOOD TO KNOW

Used for counting events in a fixed period when events are rare and independent.

When
  • Counting events per time/space
  • Events are rare
  • Events don't affect each other
Where (DA)
  • Orders per hour
  • Support tickets per day
  • Server errors per minute
  • Customer arrivals per hour
Why
  • Models frequency of rare events
  • Used in anomaly detection
How to Identify
  • Question involves "how many per..."
  • Count data (0,1,2,3...)
P(X = k) = (λ^k × e^-λ) ÷ k!    where λ = average rate
python from scipy.stats import poisson # Average 5 support tickets per hour (λ = 5) # What's the probability of getting exactly 8 tickets in 1 hour? p = poisson.pmf(k=8, mu=5) print(f"P(exactly 8 tickets) = {p:.4f}") # P(more than 10 tickets) - could indicate unusual activity p_overflow = 1 - poisson.cdf(k=10, mu=5) print(f"P(>10 tickets) = {p_overflow:.4f}")
🗺️ 2.4 — Distribution Decision Map USE THIS
What type of data do you have? │ ├── Continuous data (heights, ages, prices)? │ ├── Is it symmetric (bell-shaped)? │ │ ├── YES → Normal Distribution (use T-test, ANOVA, Pearson) │ │ └── NO (skewed like Fare) → Log-Normal / Transform it (use Mann-Whitney, Spearman) │ ├── Binary outcome (0/1, Yes/No)? │ └── Binomial Distribution (Survived, Clicked, Churned) │ ├── Counting events per time/space? │ └── Poisson Distribution (orders/hour, errors/day) │ └── Running statistical tests? ├── Chi-Square test → uses Chi-Square Distribution ├── T-test → uses T Distribution └── ANOVA → uses F Distribution

Hypothesis Testing
The backbone of every statistical test you'll ever run
🎓
Think of it like a courtroom: The null hypothesis (H0) is "innocent until proven guilty." You need strong evidence (low p-value) to reject it and declare the pattern is real.
⚖️ 3.1 — H0 and H1 FOUNDATION
H0 — Null Hypothesis

"Nothing is happening. There is NO relationship, NO difference."


Titanic examples:

  • Sex has NO effect on survival
  • Mean age of survivors = mean age of non-survivors
  • Pclass and Survival are independent
H1 — Alternate Hypothesis

"Something IS happening. There IS a real relationship or difference."


Titanic examples:

  • Sex significantly affects survival
  • Survivors are significantly younger/older
  • Pclass and Survival ARE related
📉 3.2 — The p-value CRITICAL

The p-value answers: "If H0 were true (no real effect), how likely is it that I'd see this data or something even more extreme just by random chance?"

A very small p-value means: "This is very unlikely to be random. Something real is happening."

p-value = the area in the tail(s) beyond your test statistic. Smaller area = more extreme = more significant.
p-valueMeaningDecision
p < 0.001Extremely strong evidence against H0✅ Reject H0 confidently
p < 0.01Very strong evidence✅ Reject H0
p < 0.05Standard threshold — significant✅ Reject H0
p > 0.05Not enough evidence❌ Fail to reject H0
⚠️
Common mistake: p-value is NOT the probability that H0 is true. It's the probability of seeing this data IF H0 were true. These are different things!
🎚️ 3.3 — Significance Level (α) and Confidence Level MUST KNOW

α (alpha) = your threshold for rejecting H0. Usually 0.05.

You set this BEFORE running the test.

α = 0.05 means: you accept a 5% risk of being wrong when you reject H0

This is called the "false positive rate" or Type I error rate.

Confidence Level = 1 - α

If α = 0.05 → 95% confidence level

95% Confidence = 95% sure the pattern is real, not random

In real DA work, 0.05 is the standard unless the domain demands higher (medical = 0.01, finance = 0.01).

❌ 3.4 — Type I & Type II Errors KNOW FOR INTERVIEWS
Error TypeWhat HappenedAlso CalledTitanic Example
Type I (False Positive) Rejected H0 but it was actually true α error Concluded Sex affects survival — but actually it was random luck in this sample
Type II (False Negative) Didn't reject H0 but it was actually false β error Concluded Age doesn't matter — but actually it does, your sample was too small to detect it

Normality Testing
Check this BEFORE any test. It decides your entire path forward.
🚦
Rule: T-test and ANOVA assume your data is normally distributed. If you skip normality checking and use T-test on skewed data (like Fare), your results will be wrong. Always check first.
🔍 4.1 — How to Check Normality STEP BY STEP

Method 1: Visual Check (Quick)

python import matplotlib.pyplot as plt import scipy.stats as stats # Histogram + KDE — should look bell-shaped sns.histplot(df['Age'], kde=True) plt.title('Age Distribution — is it bell-shaped?') plt.show() # Q-Q Plot — points should follow straight diagonal line stats.probplot(df['Age'], dist="norm", plot=plt) plt.title('Q-Q Plot for Age') plt.show()

Method 2: Shapiro-Wilk Test (Statistical)

python from scipy.stats import shapiro # H0: The data IS normally distributed # H1: The data is NOT normally distributed stat, p = shapiro(df['Age']) print(f"Shapiro-Wilk stat: {stat:.4f}") print(f"p-value: {p:.4f}") if p > 0.05: print("✅ Normal — Use T-test / ANOVA / Pearson") else: print("❌ Not Normal — Use Mann-Whitney / Kruskal-Wallis / Spearman")
📌
Shapiro-Wilk p-value logic is REVERSED:
p > 0.05 → Fail to reject H0 → Data IS normal ✅
p < 0.05 → Reject H0 → Data is NOT normal ❌
Remember: here we WANT a high p-value!

Method 3: Skewness Rule of Thumb

Skewness ValueWhat it meansNormal?
-0.5 to +0.5Approximately symmetric✅ Treat as normal
-1 to -0.5 or 0.5 to 1Moderately skewed⚠️ Use caution
< -1 or > 1Highly skewed❌ Not normal — use non-parametric
🚢
Your Titanic data:
Age skewness = 0.53 → Moderate → T-test is borderline acceptable (n=891 is large, so okay)
Fare skewness = 4.78 → Extremely skewed → ALWAYS use Mann-Whitney for Fare, never T-test

Statistical Tests
The tools you actually run — when, why, where, and how
χ² 5.1 — Chi-Square Test CAT vs CAT
When
  • Both variables are categorical
  • Sample size ≥ 30
  • Expected freq in each cell ≥ 5
Where (DA)
  • Sex vs Survived
  • Pclass vs Survived
  • Embarked vs Survived
  • Any cat col vs target
Why
  • Tests if two categories are independent
  • Validates your crosstab findings
Assumptions
  • Observations are independent
  • Expected cell count ≥ 5
  • Data is categorical
🔬
H0: Sex and Survival are INDEPENDENT (no relationship)
H1: Sex and Survival are DEPENDENT (relationship exists)
python from scipy.stats import chi2_contingency # ── Sex vs Survived ────────────────────────────── ct = pd.crosstab(df['Sex'], df['Survived']) chi2, p, dof, expected = chi2_contingency(ct) print(f"Chi2 statistic : {chi2:.4f}") print(f"p-value : {p:.6f}") print(f"Degrees of freedom: {dof}") if p < 0.05: print("✅ SIGNIFICANT — Sex IS related to Survival") else: print("❌ NOT SIGNIFICANT — relationship could be random") # Run for all categorical columns: for col in ['Sex', 'Pclass', 'Embarked', 'Cabin_known']: ct = pd.crosstab(df[col], df['Survived']) _, p, _, _ = chi2_contingency(ct) status = "✅ Significant" if p < 0.05 else "❌ Not Significant" print(f"{col:15} p={p:.6f} {status}")
📊
Cramér's V — After Chi-Square, measure HOW STRONG the relationship is (not just whether it exists):
V = sqrt(chi2 / (n × min(r-1, c-1)))
V near 0 = weak, V near 1 = strong. Chi-square only gives significance, not strength.
📊 5.2 — T-Test NUM vs 2 GROUPS

Compares the means of a numerical column between two groups. Tests whether the difference in means is real or could be random.

TypeWhen to UseTitanic Example
Independent (2-sample)2 different unrelated groupsAge of survivors vs non-survivors
PairedSame people measured twiceBefore/after treatment in medical data
One-sampleYour sample vs a known valueIs Titanic avg age = global avg age of 1912 (30)?
🔬
H0: Mean age of survivors = Mean age of non-survivors
H1: Mean age of survivors ≠ Mean age of non-survivors
python from scipy.stats import ttest_ind, ttest_1samp, ttest_rel # ── Independent T-test: Age vs Survival ────────── survived = df[df['Survived']==1]['Age'] not_survived = df[df['Survived']==0]['Age'] t_stat, p = ttest_ind(survived, not_survived) print(f"t-statistic: {t_stat:.4f}") print(f"p-value : {p:.4f}") print(f"Mean Age (survived) : {survived.mean():.2f}") print(f"Mean Age (not survived) : {not_survived.mean():.2f}") # ── One-sample T-test ───────────────────────────── # Is Titanic avg age = 30 (known 1912 average)? t, p = ttest_1samp(df['Age'], popmean=30) print(f"One-sample p: {p:.4f}")
⚠️
Check normality FIRST! If data is not normal, don't use T-test. Use Mann-Whitney U instead. (Fare is skewed → always use Mann-Whitney for Fare)
🔀 5.3 — Mann-Whitney U Test NON-PARAMETRIC T-TEST

The non-parametric alternative to T-test. Use when your data is NOT normally distributed.

Instead of comparing means, it compares rankings/medians — much more robust for skewed data.

When
  • Data is NOT normal (skewed)
  • Comparing 2 groups
  • Ordinal data
Where (DA)
  • Fare vs Survived (skewed!)
  • Any skewed numerical vs groups
  • Likert scale comparisons
Why
  • Works without normality assumption
  • Resistant to outliers
Rule of Thumb
  • Skewness > 1 → use this
  • When T-test gives weird results
python from scipy.stats import mannwhitneyu # ── Fare vs Survival (skewed data) ─────────────── fare_survived = df[df['Survived']==1]['Fare'] fare_not_survived = df[df['Survived']==0]['Fare'] u_stat, p = mannwhitneyu(fare_survived, fare_not_survived, alternative='two-sided') print(f"U statistic : {u_stat:.2f}") print(f"p-value : {p:.6f}") print(f"Median Fare (survived) : {fare_survived.median():.2f}") print(f"Median Fare (not survived) : {fare_not_survived.median():.2f}")
📊 5.4 — ANOVA (Analysis of Variance) NUM vs 3+ GROUPS

ANOVA extends the T-test to 3 or more groups. It tests whether the mean of a numerical variable is significantly different across multiple groups.

🔑
Key point: ANOVA only tells you "at least one group is different." It does NOT tell you which specific groups differ. For that, you need a Post-hoc test (Tukey's HSD).
🔬
H0: Mean Fare is the same across all 3 Pclasses
H1: At least one Pclass has a significantly different mean Fare
python from scipy.stats import f_oneway from statsmodels.stats.multicomp import pairwise_tukeyhsd # ── ANOVA: Fare across Pclass ──────────────────── c1 = df[df['Pclass']==1]['Fare'] c2 = df[df['Pclass']==2]['Fare'] c3 = df[df['Pclass']==3]['Fare'] f_stat, p = f_oneway(c1, c2, c3) print(f"F-statistic : {f_stat:.4f}") print(f"p-value : {p:.6f}") # ── Tukey Post-hoc: WHICH classes differ? ──────── tukey = pairwise_tukeyhsd( endog=df['Fare'], groups=df['Pclass'], alpha=0.05 ) print(tukey) # Shows exactly which pairs differ # ── Non-parametric alternative (Fare is skewed) ── from scipy.stats import kruskal h_stat, p = kruskal(c1, c2, c3) print(f"Kruskal-Wallis p: {p:.6f}")
🔗 5.5 — Correlation Test (Pearson & Spearman) NUM vs NUM

You already calculated correlations in Phase 3B (heatmap). But those were just numbers — you never tested if those correlations are statistically significant.

TestWhenAssumes Normal?Titanic Use
Pearson Both variables are continuous & normal, linear relationship YES Age vs Survived (Age is approximately normal)
Spearman Non-normal, ordinal, or non-linear data NO Pclass vs Survived, Fare vs Survived (skewed)
Pearson r: strength AND direction of LINEAR relationship
Spearman ρ: strength of MONOTONIC relationship (as X goes up, Y goes up/down)
python from scipy.stats import pearsonr, spearmanr # ── Pearson: Age vs Survived ────────────────────── r, p = pearsonr(df['Age'], df['Survived']) print(f"Pearson r = {r:.4f}, p = {p:.4f}") # ── Spearman: Pclass vs Survived (ordinal) ─────── rho, p = spearmanr(df['Pclass'], df['Survived']) print(f"Spearman ρ = {rho:.4f}, p = {p:.4f}") # ── Spearman: Fare vs Survived (skewed) ────────── rho, p = spearmanr(df['Fare'], df['Survived']) print(f"Spearman ρ = {rho:.4f}, p = {p:.4f}")

📏 5.6 — Confidence Intervals FOR REPORTING

A p-value tells you IF something is significant. A confidence interval tells you HOW BIG the effect actually is and in which direction.

📌
"The average age of Titanic survivors is 28.4 years, 95% CI [26.8, 30.0]"

This means: you're 95% confident the TRUE average age of survivors (in the population this represents) lies between 26.8 and 30.0 years.
python import numpy as np from scipy import stats def confidence_interval(data, confidence=0.95): n = len(data) mean = np.mean(data) se = stats.sem(data) # standard error of mean ci = stats.t.interval(confidence, df=n-1, loc=mean, scale=se) return mean, ci # CI for survivor ages mean, (low, high) = confidence_interval(df[df['Survived']==1]['Age']) print(f"Survivors — Mean Age: {mean:.1f}, 95% CI: [{low:.1f}, {high:.1f}]") # CI for non-survivor ages mean, (low, high) = confidence_interval(df[df['Survived']==0]['Age']) print(f"Non-Survivors — Mean Age: {mean:.1f}, 95% CI: [{low:.1f}, {high:.1f}]")
✅
How to read overlapping CIs:
If the CIs of two groups DON'T overlap → difference is likely significant.
If they DO overlap → likely not significant (but run the T-test to confirm).

🧠 Golden Rules Every DA Must Remember
1️⃣
Always check normality FIRST before choosing a test. Skewness >1 → non-parametric.
2️⃣
p < 0.05 = significant. But small p doesn't mean big effect. Always check effect size too.
3️⃣
Chi-Square for categorical, T-test for numerical vs 2 groups, ANOVA for 3+ groups.
4️⃣
Correlation ≠ Causation. Fare correlates with survival, but high fare didn't cause survival.
5️⃣
ANOVA just says someone is different. Run Tukey HSD after to find out who.
6️⃣
Confidence Intervals are more informative than p-values for reporting. Use both.
